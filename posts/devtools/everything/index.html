Everything the tool you need

Everything (Realtime filesystem indexing)

Introduction

Everything is a tool that query NTFS filesystem for any change that happen to any referenced device and provide a way to have control over you machine. the binary it self is no more than 2 megs. and it can change the way you interact with windows explorer forever.

If you combine the tool with a hotkey you can decrease the use of windows explorer to 10%.

I’m such lazy person who doesn’t want to memorize in my poor brain my subsystem’s files location, So what I do is only make sure to put the files in a adequate location and forget about them. I memorize only the file/folder name that I want to open, or just a file under the folder that I want to search for.

This tool becomes very handy if you are a developer and you want to access files very often. By time, you will discover how useful this tool is, I don’t say these words without proofs. I have been using this tool for years now, since Windows XP era, and my cogent argument is that I’m still using it every hour (even when i’m sleeping :@ ).

Search options & features

  • Advanced Realtime search
  • Remote Realtime web browser search using mini Http server
  • Remote Realtime search via mini Ftp server

Searching for files requires often the use of regular and logical expression, I advice every one to read the help at least once, see the image below to have an idea about it.

Use Cases

  • search for a specific file
  • search for a pattern ‘*.dll, *.exe, …’
  • search for a pattern under a location
  • search the count of pattern
  • search under a specific location

Use case example (where is my config file)

Suppose that you have just installed XMAPP or anything similar that includes many other programs, like apache server, mysql, php etc… and you want to know in which location your config files are.

For example, you are looking for mysql.ini, but there are many files with the same name to the point that will be confused.

Everything can help, to make this problem easier, instead of opening these files one by one to know which one is the actual config file, you only need to search for mysql.ini under xampp folder, you can open them faster, because you will see them all in one location. you will see their path, size and modification date.

Example: you want to search for the host file under system

type in the search field : \system hosts

if you want to filter more add !*.ics !*.sam and you will have only your file. note that ! will negate the result, so !*.ics means exclude all files that have the extension ics

The final result should be \system hosts !*.ics !*.sam.

Define a key for better productivity

After installing everything, it will be better to configure a hotkey to launch everything search window instead of going to try menu and select the icon.

To configure it, just :

  • Go to to options
  • Specify a key in (new window hotkey)
  • validate by clicking on OK
